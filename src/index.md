Find the right primitive, then compose the elegant thing on top. That's the whole
trick — and I'll chase it further than is strictly reasonable. Far enough that I
wrote Perl a language server from scratch, in Rust, because I wanted one that
genuinely *understood* the code: type inference, cross-file resolution, the works.
[Here's how that went.](/projects/)

It's a pattern. I also created and maintain
[tree-sitter-perl](https://github.com/tree-sitter-perl/tree-sitter-perl) — the
Perl grammar behind Zed, Neovim, Helix, and Emacs, and the composable primitive a
lot of other tools quietly build on.
